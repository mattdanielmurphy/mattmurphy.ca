# Goal: Dynamic Markdown Notes Feature

Added the ability to publish public markdown notes directly from a private GitHub repository (`mattdanielmurphy/personal-notes`).

## Changes Made
- Installed `gray-matter` for parsing frontmatter and `marked` for rendering Markdown to HTML.
- Created `src/styles/Notes.module.css` with clean, readable typography for markdown content.
- Added `src/pages/notes/index.js` to fetch all public markdown files from the repository using the GitHub Git Trees REST API and concurrent fetching. Uses ISR (`getStaticProps` with revalidation) to avoid rate limits.
- Added `src/pages/notes/[...slug].js` as a catch-all route to handle notes in subfolders (e.g. `/notes/folder/my-note`). It fetches the raw markdown content, parses frontmatter to ensure `public: true`, and renders HTML.

## What Worked
- Fetching dynamically via Next.js ISR allows the site to stay static while receiving updates behind the scenes without rebuilds.
- `[...slug].js` elegantly supports arbitrarily nested folders in the Obsidian vault.

## What Didn't Work / Known Issues
- Currently, generating the index page (`/notes`) requires fetching *all* markdown files in the repo to check frontmatter. As the vault grows to thousands of notes, this could exhaust the 5,000 req/hr GitHub rate limit. Caching is currently set to 5 minutes (`revalidate: 300`) to mitigate this.

## Architecture Notes
- The site relies on a GitHub Personal Access Token (`GITHUB_PAT`) being present in the environment variables to read the private repository.
- Incremental Static Regeneration (ISR) is used instead of Server-Side Rendering (SSR) so that page loads remain instantly fast, and rate limits are respected.
